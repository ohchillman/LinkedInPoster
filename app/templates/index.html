LinkedIn Poster API

Информация об API

Этот сервис позволяет публиковать контент в LinkedIn через API. Вы можете отправлять запросы на /post с необходимыми параметрами.

Требования к токену доступа LinkedIn

Для корректной работы API, ваш токен доступа LinkedIn должен иметь следующие разрешения:

  • w_member_social - для публикации постов

Если вы получаете ошибки авторизации, убедитесь, что ваш токен:

  • Не истек (токены LinkedIn обычно действительны 60 дней)
  • Имеет все необходимые разрешения
  • Получен для правильного приложения LinkedIn

Перейдите в консоль разработчика LinkedIn для управления вашими приложениями и токенами.

Новая функция: Указание ID пользователя LinkedIn

Если у вас возникают проблемы с доступом к профилю через API LinkedIn, вы можете указать ваш ID пользователя LinkedIn напрямую:

  • Параметр user_id позволяет обойти необходимость запроса к профилю
  • Это особенно полезно, если ваш токен не имеет разрешения r_liteprofile

Ваш ID пользователя LinkedIn можно найти в URL вашего профиля или в консоли разработчика.

Пример запроса:

POST /post
Content-Type: multipart/form-data

linkedin_client_id: YOUR_CLIENT_ID
linkedin_client_secret: YOUR_CLIENT_SECRET
linkedin_access_token: YOUR_ACCESS_TOKEN
text: Текст вашего поста
images: [файлы изображений]
proxy_host: (опционально) хост прокси
proxy_port: (опционально) порт прокси
proxy_username: (опционально) имя пользователя прокси
proxy_password: (опционально) пароль прокси
user_id: (опционально) ваш ID пользователя LinkedIn

Пример успешного ответа:

{
    "status": "success",
    "message": "Пост успешно опубликован",
    "post_url": "https://www.linkedin.com/feed/update/urn:li:share:1234567890"
}

Пример ответа с ошибкой:

{
    "detail": "Не удалось создать пост в LinkedIn. Убедитесь, что токен доступа действителен и имеет разрешение w_member_social."
}

Важная информация о токене LinkedIn

Для успешной публикации постов, убедитесь, что ваш токен доступа имеет разрешение w_member_social.
